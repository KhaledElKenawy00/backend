# Room Service — Backend Summary

## Overview

A Spring Boot microservice that coordinates Discord-style **voice/video rooms**. Runs on port **8086**. It owns rooms, membership, and live presence; delegates the room's **text chat** to the Chat Service (over RabbitMQ); and assigns the **Agora channel name** each client joins. Media itself is handled by Agora directly from the frontend — the backend never touches it, and in v1 does not mint Agora tokens.

Full design: [`Docs/room-service-arc.md`](../../Docs/room-service-arc.md). Frontend/API contract: [`frontend.md`](./frontend.md).

## Tech Stack

| Component | Tech |
|---|---|
| Framework | Spring Boot 4.0.5, Java 21 |
| Database | MongoDB (rooms) — shared instance, DB `room_service` |
| Cache | Redis (presence + WS tickets) — shared instance, keys prefixed `room:*` |
| Messaging | RabbitMQ — **publishes** room-channel events to chat-service (shared broker) |
| Realtime | STOMP over WebSocket |
| Media | Agora (frontend SDK; backend assigns the channel name only) |
| Build | Maven (wrapper included) |

## Infrastructure ownership (brings up nothing of its own)

| Infra | Owner | Usage |
|---|---|---|
| MongoDB | `chat-service/docker-compose.yml` | DB `room_service` (chat uses `chat_service`; same instance) |
| Redis | `chat-service/docker-compose.yml` | presence hashes + WS tickets (`room:*`) |
| RabbitMQ | `user-service` | publish room-channel events |

Start order: **user-service (RabbitMQ) → chat-service (Mongo/Redis) → room-service**.

> **Mongo credentials:** provided by an explicit `MongoConnectionDetails` bean in `MongoConfig` (parsed from `spring.data.mongodb.uri`). On Spring Boot 4 without the docker-compose service-connection integration, credentials embedded in the connection string were not being applied (the connection came through unauthenticated); the explicit bean fixes that.

## Architecture

```
Client → Nginx (JWT validation) → Room Service (trusts X-User-* headers)
Room Service ──(RabbitMQ: room.exchange / room.channel.event)──▶ Chat Service  (creates/syncs ROOM channels)
Frontend ──────────────────────(Agora SDK, by channel name)──────────────────▶ Agora cloud
```

## Data Model

### Room (MongoDB)
- `id` (ObjectId) — auto-generated
- `workspaceId` — workspace the room belongs to
- `name` — display name
- `channelId` — bound Chat Service `ROOM` channel (generated by Room Service, sent in the create event)
- `agoraChannelName` — `room-{id}` — the Agora channel every client joins
- `members` — user IDs allowed to see/enter the room (mirrored to the chat channel)
- `maxParticipants` — concurrent-participant cap (default 25)
- `createdBy`, `createdAt`, `updatedAt`

Indexes (auto-created): unique `(workspaceId, name)`, unique `channelId`, multikey `members`.

### Presence (Redis, ephemeral)
- `room:{roomId}:participants` (hash): field = userId, value = `muted,cameraOn,screenSharing,joinedAtEpochMs`
- `room:{roomId}:alive:{userId}` (TTL ~30s): liveness backstop; stale entries are reaped lazily on read

## Key Features

### Rooms
- Create (generates `channelId` + `agoraChannelName`, publishes `ROOM_CHANNEL_CREATE`; `409` on duplicate name; `503` + room rollback if the broker is down)
- List the caller's rooms in a workspace (private — membership-filtered)
- Get / rename / delete (delete also removes the bound chat channel, clears presence, broadcasts `ROOM_CLOSED`)
- Add / remove members (mirrored to the bound chat channel)

### Presence & signalling
- Join records presence with an **atomic capacity check** (Lua: `HEXISTS`/`HLEN`/`HSET`) → `409` when full; join is idempotent
- Leave / participants; live voice-state (mute/camera/screen-share) via STOMP `/app/room/state`; `/app/room/heartbeat` refreshes liveness
- Broadcasts to `/topic/room/{id}`: `PARTICIPANT_JOINED`, `PARTICIPANT_LEFT`, `STATE_CHANGED`, `ROOM_UPDATED`, `ROOM_CLOSED`
- WebSocket ticket auth (one-time, 60s); subscription is membership-gated; clients may only `SEND` to `/app/**`

### Chat integration (RabbitMQ producer)
- Publishes `RoomChannelEvent` to `room.exchange` with routing key `room.channel.event`:
  `ROOM_CHANNEL_CREATE` / `ROOM_CHANNEL_DELETE` / `ROOM_CHANNEL_ADD_MEMBER` / `ROOM_CHANNEL_REMOVE_MEMBER`
- Chat Service consumes these and creates/deletes/syncs the bound `ROOM` channel (idempotent; the create carries the Room-supplied `channelId`)
- Eventual consistency: the chat channel materializes shortly after room creation

### Agora (v1)
- App-ID-only; the backend assigns/returns `agoraChannelName` only (no token minting). v2: server-side tokens with the App Certificate.

## How to Run

```bash
export JAVA_HOME=/path/to/jdk21      # Windows: $env:JAVA_HOME = "C:\path\to\jdk21"
# user-service + chat-service must be running first (RabbitMQ, Mongo, Redis)
./mvnw spring-boot:run               # Windows: mvnw.cmd spring-boot:run
```
Starts on `http://localhost:8086`. Room Service owns no Docker infra — it connects to the shared Mongo/Redis (chat-service) and RabbitMQ (user-service).
